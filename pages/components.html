Components

Components are the reusable building blocks of the California State Template.

Each component is designed to enhance user experiences, and has been created to work together to create patterns.

Our components are:

  • Created with performance and accessibility in mind
  • Usable in any framework

All components

Use these components to present information and images.

Accordion

An accordion is a list of headings that hide or reveal additional content when selected.

Alert

An alert keeps users informed of important and sometimes time-sensitive changes.

Blockquote

A blockquote emphasizes important speaker quotes or content excerpts.

Breadcrumb

A breadcrumb indicates the current page's location within a navigational hierarchy.

Button

A button draws attention to your primary call to action with a large clickable surface.

Card

A card displays a snippet of information and can link to more details upon interaction.

Countdown timer

A countdown timer emphasizes an important event by counting toward or away from a fixed point in time.

Form elements

A form allows visitors to enter and submit information to a server.

Horizontal separator

The horizontal separator is a style component that uses a line to separate content into sections.

Lists

A list is a vertical grouping of related content.

Modal

A modal displays content in a temporary window that appears over the page content

Number counter

The number counter adds animation to numbers.

Parallax

The parallax is a visual design component. When scrolling, it allows background and foreground elements to move at different speeds.

Progress tracker

A progress tracker displays the steps and progress through a journey.

Search bar

The search bar allows visitors to search for specific content using keywords or phrases.

Table

A table displays large amounts of data in rows and columns.

Tabs

Tabs organize related content into panels and enable visitors to navigate between them.

Additional Bootstrap CSS & Components

We strongly recommend using the documented State Web Template components and patterns whenever possible; however, for more CSS classes and options as well as components we suggest utilizing what is available in Bootstrap first, before developing your own custom code.

NOTE: When using custom components and content modules that are not documented as part of the State Template, migration to future versions of the State Template may require additional work.
